# EPOS SYSTEM OVERVIEW

## NAME
Evolution Power OS (EPOS)

## PURPOSE
EPOS is an internal operational system for Evolution Power.

It must centralize internal workflows inside WordPress using WPCode snippets only.

## CORE PRINCIPLE
EPOS is not a traditional software application.

It is a modular internal system composed of independent WPCode snippets running inside WordPress.

## MAIN TECHNICAL BASE
- WordPress
- WPCode
- PHP
- WordPress REST API
- MySQL through WordPress database access

## WHAT EPOS MUST NOT BECOME
EPOS must not become:
- a React app
- a Node app
- a Laravel app
- a plugin-based architecture
- a folder-based frontend system
- a disconnected multi-environment product

## FUNCTIONAL VISION
EPOS should evolve into a unified internal dashboard where operations are managed in one place.

## DEVELOPMENT RULE
All future implementation must respect the master rules file and the mandatory snippet architecture.
